### 最直接的解决办法：使用 `tar1090` (目前最流行的 Web UI)

`tar1090` 是目前 ADS-B 圈子里公认最强的网页界面，它可以直接连接你本地运行的 `dump1090` 原始数据。

#### 1. 克隆前端代码

找个你喜欢的目录（比如 `~/Documents`），把这个开源的前端项目拉下来：

Bash

```
git clone https://github.com/wiedehopf/tar1090.git
```

#### 2. 配置数据源

这个前端需要读取 `dump1090` 生成的 `aircraft.json`。

- 首先，重启你的 `dump1090`，让它把数据写到一个固定的文件夹：
    
    Bash
    
    ```
    # 创建数据存放目录
    mkdir -p ~/adsb_web_data
    # 运行并写入数据
    dump1090 --interactive --net --write-json ~/adsb_web_data
    ```
    
- 然后，在 `tar1090` 文件夹里，把它的数据目录软链接到你刚才创建的目录：
    
    Bash
    
    ```
    cd tar1090/html
    ln -s ~/adsb_web_data data
    ```
    

#### 3. 启动本地 Web 服务

在 `tar1090/html` 目录下，用 Python 启动服务器：

Bash

```
python3 -m http.server 8080
```

#### 4. 看图

打开浏览器访问：`http://localhost:8080`。你就能看到那个带地图、带轨迹、带侧边栏的完整雷达界面了！

---

### 为什么这个能解决问题？

`dump1090` 本身其实是个“哑巴后端”，它只负责从无线电里抠出数据。正常的网页图渲染流程是这样的：

1. **后端**：`dump1090` 监听 USB 里的 RTL-SDR，把飞机坐标写进 `aircraft.json`。
    
2. **中转**：Python 服务器把这个 `.json` 文件通过 HTTP 协议发出去。
    
3. **前端**：`tar1090`（或者原版 HTML）通过 Javascript 把这些坐标点画在地图上。
